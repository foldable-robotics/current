    A03: Dynamics

    Introduction

    The purpose of this assignment is to acquaint you with the concepts of creating and using a dynamic simulation for answering questions

    Instructions

    Part 1: Constraints revisited

    The purpose of this exercise is to get experience working with constraints in MuJoCo, and how the parameters work. You will create a planar four-bar linkage in mujoco, adapted from the double pendulum code shared in the dynamics 1 handout.

    1. Define a total of five links in series with each other. From proximal to distal they should be labeled "A", "B", "C", "D", and "Ap". "A" should be grounded and the other four should initially align with the x asis of your A frame.
      • Ensure your xml is in "degree" mode
      • Add a small amount of damping to each joint (damping=.1)
      • set the mass of each body to 1 (defining mass inside the <geom> is sufficient)
      • all the links should be a length of 1 for this exercise.
      • You can ground your \(A\) link by removing the first hinge from the kinematic chain.
      • Disable contact in your simulation. This will help body A and Ap move through each other in order to establish the right constraints.

    2. Add the following code to the bottom of the xml, right above the </mujoco> line. What happens? Does this look like a four-bar?

      <equality>
    <weld name="weld1" active="true" body1="A" body2="Ap" relpose="0 0 0 1 0 0 0" anchor="0 0 0" />
</equality>

    3. change axisangle="0 1 0 0" to axisangle="0 1 0 -80" for bodies "B" thru "Ap". Display the pose at t=0.

      1. Report the joint angles at t=0. Based on our in-class discussion, explain why the values are what they are.
      2. Show the 3D rendering at t=3.
      3. Now describe what happens when you run the step through the simulation. How is this different from your first try?

    4. Finally, revert axisangle="0 1 0 -80" to axisangle="0 1 0 0" for bodies "B" thru "Ap". After the line that performs a resetData(), insert the following line:

      data.qpos = numpy.array([-math.pi*80/180,-math.pi*80/180,-math.pi*80/180,-math.pi*80/180])

      Describe what happens

    5. Play with the relpose and anchor terms. What happens when you use:

      1. relpose="1 1 1 1 0 0 0" anchor="1 1 1"
      2. relpose=".5 0 0 1 0 0 0" anchor="0 0 0"
      3. relpose="0 0 0 .866 0 -.5 0" anchor="0 0 0"
      4. relpose="0 0 0 .866 0 -.5 0" anchor=".5 0 0"

      Supply an image at t=3 for each of these cases, and explain in your own words what you see for each case. Using these examples, please explain what the first three terms of relpose represent, what the last four terms of relpose represent, and what the three terms of anchor represent.

    Part 2: Templates

    The purpose of this exercise is to learn how to re-run simulations repeatedly with different model parameters. You will be using this approach to find match the values in your simulation to this dataset.

    If you have not yet, refresh your knowledge of string formatting in Python.

    • https://docs.python.org/3/library/stdtypes.html#str.format
    • https://docs.python.org/3/library/string.html#formatstrings

    Then, using the double pendulum code shared in the dynamics 1 handout as a starting point,

    1. Ensure the following parameters are set:
      1. Mass of each body is 1
      2. Timestep is set to .001
      3. Stiffness and damping (parameters k and b) are defined for each joint to be 10 and 1, respectively.
      4. Run the code and plot the joint positions over three seconds.
    2. Now, modify the xml to act as a template variable, so that all the damper and spring values in the model can be replaced by two variables, b and k, respectively.
      1. Generate the xml variable from the template variable using the python format() function, with k=10 and b=1. Re-run the simulation and confirm that the output matches your original results.
    3. Load the dataset and check the initial values of q and w. Set the initial values of your simulation to match the dataset.
    4. Move the code required to define and run a single simulation into a function that takes two parameters: b and k. It should output two variables: t and q, your time vector and your joint vector.
    5. Calculate the mean squared error between the joint position vector q and the value for q found in the dataset over each time step from t=0s to 3s.

    6. Create a for loop that iterates over

      \[(k_i,b_j), \text{ } k_i=10,20,30,\ldots,100, \text{ }b_j=1,\ldots,10\]

      and finds the total error over the whole simulation run per stiffness and damping \((k_i,b_j)\) pair.

    7. Plot the error as a function of \(k_i\) and \(b_j\), and identify the values of \(k_i\) and \(b_j\) that result in the smallest error.

    8. What have we just done by hand? What steps would you have to do to automate this better using built-in functions already provided in Python or its packages?
